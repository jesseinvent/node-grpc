# gRPC

## Pros of gRPC:
- Uses HTTP2 & Protocol buffers
- Fast & compact because data are sent in protocol buffers (binary format)
- One Client Library per langauge
- Progress Feedback (upload)
- Cancel Request (H2)

## Cons of gRPC:
- Schema retrictions
- Thick Client
- Low support
- Poor error handling
- No native browser support
- No timeouts (pub/sub)
